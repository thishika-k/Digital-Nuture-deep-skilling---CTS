# Hands-On 1: QA Concepts and Software Testing Fundamentals

## Objective

Understand the fundamentals of software testing, software development lifecycle, testing lifecycle, testing levels, testing types, and defect management.

---

## Software Testing

Software testing is the process of verifying and validating that an application works as expected and meets user requirements. It helps identify defects before the software is released.

---

## Quality Assurance (QA) vs Quality Control (QC)

| Quality Assurance (QA)                     | Quality Control (QC)                 |
| ------------------------------------------ | ------------------------------------ |
| Process-oriented                           | Product-oriented                     |
| Prevents defects                           | Detects defects                      |
| Focuses on improving development processes | Focuses on testing the final product |
| Performed throughout development           | Performed after development          |

---

## Software Development Life Cycle (SDLC)

The Software Development Life Cycle defines the phases involved in developing software.

1. Requirement Analysis
2. Planning
3. Design
4. Development
5. Testing
6. Deployment
7. Maintenance

---

## Software Testing Life Cycle (STLC)

The Software Testing Life Cycle defines the testing activities performed during software development.

1. Requirement Analysis
2. Test Planning
3. Test Case Development
4. Test Environment Setup
5. Test Execution
6. Defect Reporting
7. Test Closure

---

## Levels of Testing

* Unit Testing
* Integration Testing
* System Testing
* Acceptance Testing (UAT)

---

## Types of Testing

### Functional Testing

Verifies that the application functions according to business requirements.

Examples:

* Smoke Testing
* Sanity Testing
* Regression Testing
* Integration Testing
* System Testing

### Non-Functional Testing

Evaluates the quality attributes of the application.

Examples:

* Performance Testing
* Load Testing
* Stress Testing
* Security Testing
* Usability Testing
* Compatibility Testing

---

## Defect Life Cycle

Typical defect states:

New → Assigned → Open → Fixed → Retest → Closed

If the issue still exists after retesting, the defect is reopened and fixed again.
