# Hands-On 7: Page Object Model (POM)

## Objective
Implement the Page Object Model (POM) design pattern to improve the maintainability and reusability of Selenium automation scripts.

## Technologies Used
- Python
- Selenium WebDriver
- Pytest

## Tasks Performed
- Created Page Object classes
- Separated page elements from test cases
- Reused page methods across tests
- Organized project into Pages, Tests and Utilities

## Learning Outcome
- Understood the Page Object Model
- Improved code reusability
- Reduced duplication in test scripts
- Built a maintainable Selenium automation framework
